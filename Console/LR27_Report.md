# Отчёт ЛР27 — диагностика и Support Report

## Цель работы
Реализовать диагностику системы и сбор отчёта для поддержки (Support Report).

## Что сделано

### 1. Добавлен класс `DiagnosticsService`
- Расположен в `TaskTracker.App/Support/DiagnosticsService.cs`
- Собирает следующую информацию:
  - Дата и время проверки
  - Базовая директория приложения
  - Версия сборки
  - Роль пользователя и режим хранения (из config.json)
  - Существование файлов `config.json` и `tasks.json`
  - Проверка доступа и возможности записи в папки: Data, Logs, Backups, Exports, Reports
  - Количество задач: всего, активных, в корзине, в архиве
  - Наличие сегодняшних лог-файлов (`app_...log`, `errors_...log`)

### 2. Добавлены команды меню
- **28** – Диагностика (вывод в консоль)
- **29** – Собрать отчёт для поддержки (сохраняет в файл)

### 3. Реализация в `Program.cs`
- Создан экземпляр `DiagnosticsService` с передачей всех путей.
- Обработка команд 28 и 29 с выводом цветных сообщений (`ShowSuccess`, `ShowError`).
- При команде 29 автоматически создаётся папка `reports` и файл с именем `SupportReport_YYYY-MM-DD_HH-mm-ss.txt`.

### 4. Улучшения UI (по желанию)
- Цветное меню с группировкой.
- Безопасный ввод Id через `AskForId()`.
- Сообщения об успехе/ошибке/информации.
- Замена эмодзи на символы для совместимости со старыми консолями.

## Как проверить самостоятельно
1. Запустить приложение.
2. Ввести `28` – увидеть диагностику.
3. Ввести `29` – получить файл в папке `reports`.
4. Открыть файл – содержимое совпадает с выводом диагностики.
